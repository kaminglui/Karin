Profile

Each profile keeps its own watchlists, reminders, alert cooldowns, and location. Shared content (RSS feeds, market data) is the same across profiles.

Profile switch saved. Restart the bridge for all subsystems to pick it up (reminders, alerts, and feedback are still bound to the prior profile until restart).
IP routing — auto-select profile by Tailscale IP

Map each device's Tailscale IP to a profile. When a request arrives from a mapped IP, the system auto-switches to that profile (restart required to fully apply). Devices not listed fall back to the manual dropdown above.

Location

Used by the alerts panel to score signals by proximity (closer events get higher threat scores) and by tools that need a local context. City alone is the minimum useful entry; lat/lon give the threat-distance computation more precision.

Saves to the active profile's preferences.json. Alerts pick up the new location on the next refresh — no bridge restart required.

Data source API keys

Save free-tier API keys for upstream data sources here so the county-overlay populators can use them without an .env edit. Environment variables (deploy/.env) always win, so existing setups keep working. Each populator script accepts --from-prefs to read the saved value.

Keys live in the active profile's preferences.json. The Settings UI never echoes a full secret — only the last four characters as confirmation. Leave a field blank to clear it.
